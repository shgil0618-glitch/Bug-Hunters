### 🍟 “오늘 뭐 먹지?” 레시피 추천 사이트(1)
---
| 항목 | 상세 내용 |
| :--- | :--- |
| **프로젝트 목표** | Spring·MyBatis 기반의 UGC 웹 서비스 구축 (CRUD, 권한별 UI, AJAX 페이징, SQL 최적화) |
| **기간 / 인원** | 2025.11.2 ~ 2025.12.3 (3명) |
| **개발 환경 및 언어** | **Backend**: Java 11 (JDK 11), Spring Framework 4.3 (MVC, Security), MyBatis 3.5<br>**Database**: Oracle 11g<br>**Frontend**: HTML5 / CSS3, JavaScript (ES11), jQuery 3.7, Bootstrap 5 |

### 🍟 주요기능
---
```	

1. 멤버 관리 (가입/로그인/정보수정)
2. 게시판 CRUD
3. 재료 관리 및 게시글 연동
4. 페이징 · 검색 기능 → 가독성 향상
5. 권한 분리 → 보안성 강화
```

#### 🍟 트러블슈팅
---

```
<사례1>  
- 문제: HTML 화면은 정상 렌더링되나 테이블 데이터가 비어 보이는 현상 발생  
- 원인: 조회 결과가 null/empty인데도 UI에서 이를 구분하지 못함  
- 해결: 디버깅 출력으로 데이터 유무 검증, 조건부 렌더링 처리 적용  
- 성과: 페이징 파라미터와 동적 쿼리 조건을 통합 설계, 기능 간 의존성을 고려한 서버 로직 설계 강화  
- 학습: UI-서버 데이터 흐름 검증 능력 확보, 예외 상황 처리 경험 축적  
- 향후 개선 계획: 데이터 검증 로직을 Validator 모듈화하여 재사용성과 안정성을 높일 예정  

<사례2>  
- 문제: 관리 페이지에서 페이징 이동 시 모달 UI가 전체 페이지로 전환되며 구조 깨짐 발생  
- 원인: 페이지 전환 시 모달 컨텍스트 유지 실패, 전체 페이지 렌더링으로 구조 붕괴  
- 해결: 모달 내부에서 Ajax 페이징 처리, 화면 전환 없이 목록 영역만 갱신하도록 구조 개선  
- 성과: 화면·컨트롤러·쿼리를 분리해 점검하며 데이터 흐름(Model → View)을 구조적으로 검증하는 디버깅 능력 향상  
- 학습: Ajax 기반 UI 개선 및 MVC 구조 검증 경험 확보  
- 향후 개선 계획: 모달·페이징 기능을 컴포넌트화하여 재사용성과 유지보수성을 강화 예정

<사례 3>
- 문제: 대량 데이터 환경에서 복합 Join·서브쿼리로 인한 조회 속도 저하 및 페이지 로딩 지연
- 원인: ERD 설계 복잡성과 불필요한 데이터 스캔으로 성능 저하 발생
- 해결: 전체 ERD·데이터 흐름 재검토, EXISTS 조건 적용으로 불필요한 스캔 제거, 조건 기반 쿼리 최적화
- 성과: 검색 속도 개선 및 성능 안정화 달성
- 학습: SQL 최적화 기법 실무 적용 경험 확보, 대규모 데이터 환경에서도 안정적 검색 기능 구현 가능
```
