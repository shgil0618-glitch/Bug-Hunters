#### 🍟 트러블슈팅
---
```
<사례1>
- 문제: 회원 관리 페이지에서 회원정보를 불러오지 못하는 오류 발생  
- 원인: 프론트엔드 accessToken이 서버로 정상 전달되지 않아 인증 실패  
- 해결: 로컬 스토리지에서 토큰을 직접 가져와 헤더에 삽입하는 코드 추가, 
Axios 인터셉터로 토큰 자동 처리 로직 개선  
- 추후 업그레이드 계획: Refresh Token 구조 도입 및 Redis 기반 세션 관리 강화 → 장기 세션 안정성 확보  
- 성과: 인증 로직 안정화, 사용자 정보 조회 기능 정상화, 보안·UX 동시 개선  
- 학습: JWT 인증 구조와 프론트엔드-백엔드 간 토큰 전달 방식 이해, 보안·성능 균형 설계 역량 강화


<사례2>  
- 문제: 동일 사용자가 같은 게시글에 좋아요·리트윗을 반복 요청해 반응 수가 비정상적으로 증가  
- 원인: 사용자–게시글 반응 상태에 대한 서버 단 중복 제어와 데이터 무결성 제약이 부족
- 해결: ERD·데이터 흐름 재검토, 사용자 ID + 게시글 ID 기준 유니크 제약과 반응 토글 로직을 
서버에 적용
- 성과: 반응 데이터 무결성을 확보하고 SNS 인터랙션 신뢰도를 향상
- 학습: SNS 서비스에서는 UI 제어뿐 아니라 데이터 레벨에서의 상태 관리가 필수적임을 이해
```
